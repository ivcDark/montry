# Тарифы

Дата обновления: 2026-06-18.

Montry использует линейную тарифную сетку без отдельных доплат за типы мониторинга. Ограничения хранятся в `plans` и `plan_limits`, а серверная проверка выполняется через `Billing\Application\Services\LimitChecker`.

## Типы мониторинга

Базовые типы доступны на Free:

- доступность сайта (`http`);
- SSL-сертификат (`ssl`).

Все типы доступны на Pro и Team:

- доступность сайта;
- SSL-сертификат;
- срок домена;
- DNS-записи;
- `robots.txt`;
- `sitemap.xml`;
- API endpoint;
- TCP-порт.

Типы мониторинга не продаются поштучно. Подписка определяет доступный набор через лимит `allowed_monitor_types`.

## Основные лимиты

- `max_monitors` — максимальное количество активных мониторингов;
- `allowed_monitor_types.types` — доступные типы мониторинга;
- `minimum_check_interval_seconds.seconds` — минимально допустимый интервал;
- `history_retention_days.days` — срок хранения истории;
- `notification_channels.channels` — доступные каналы уведомлений;
- `can_create_projects.enabled` — возможность создавать дополнительные проекты.

Выключенные мониторинги не занимают квоту `max_monitors`.

## Где применяются ограничения

Ограничения обязательно проверяются на сервере:

- при создании мониторинга;
- при создании сайта с набором мониторингов;
- при включении или возобновлении мониторинга;
- при изменении интервала;
- при выборе типа мониторинга.

Создание сайта и его мониторингов выполняется транзакционно. Если хотя бы одна активная проверка превышает лимит или недоступна тарифу, сайт и остальные проверки не создаются.

Интерфейс также ограничивает выбор:

- недоступные типы заблокированы и ведут на страницу тарифов;
- при заполненной квоте нельзя включить ещё один мониторинг;
- варианты интервала начинаются с минимального значения текущего тарифа;
- при добавлении сайта автоматически включается максимально возможный набор доступных проверок в пределах оставшейся квоты.

Клиентские ограничения не заменяют серверные: прямой HTTP-запрос должен проходить те же проверки `LimitChecker`.
